Premise

Usually I would make a website the quick way, using a tool, such as wordpress or joomla combined with premade themes. However, for my portfolio website I decided to tackle a small challenge and build a website in 1 week from scratch. In this short post I will explain my process for creating this website in 1 week.

Learning the basics

Understand html

This is a pretty straightforward task. A single html file will define the structure and the content of a webpage. As an example the text you're reading is written inside a p element inside the html file for this post.

Understand CSS

CSS (Cascading Style Sheets) is the language used to write the style of an html page. There are plenty of properties that can be used to define certain styles and layouts, and learning all of them by heart wouldn't make much sense. Googling and using the previously mentioned resources would give a very good start on the most used properties.
In my opinion the most important stuff to understand are:

Box Model: All html elements are represented as boxes with a padding, margin, border and content.

Specificity: All css definitions can be overwritten based on the specificity (to find out how specific a definition is you can use an online calculator).

Understand Javascript

For a small scope website like this, javascript is a perfect fit for manipulating the website dynamically. This type of use case is known as DOM Manipulation.
You can modify the html by creating links to every element and apply any kind of change to the element or also remove it completely.

Bootstrap

Bootstrap is a library that uses all the previously mentioned technologies and simplifies the process of creating a website. It provides a lot of classes to simplify layout and style of an html page. The best part is that it allows to create a responsive website (good looking on every device) very easily.

Don't start with bootstrap. Make sure to have a good grasp of the other technologies first. If you start directly with bootstrap you risk not being able to modify the predefined classes to fit your needs.

As an example, this website uses bootsrap almost entirely for the responsive and layout features. All the styling is self defined using CSS. In this way it is hardly noticeable that this website is made using bootstrap.

This part was the most time consuming part. I watched the frontend part of the udemy tutorial [1] and the youtube video [2] to understand these technologies. In total I spent 4 days for this part.

Designing and implementing the website

After having a good idea of how HTML, CSS, Javascript (and optionally Bootstrap) work, you have to define the style of your website. To be entirely honest I didn't know how I wanted the website to look. I had a rough idea in mind and I started iterating over the design multiple times.

But I knew for sure that I wanted to have:

0 a minimal and responsive website

1 a dark mode (for people like me who prefer dark mode on every editor)

2 an easy way to add blog posts and have them dynamically show up

3 an option to filter the dynamically added posts using predefined tags

Achieve goal 0: minimal and responsive

I tried not having more than 3 base colors. Keeping most of the stuff minimal and without overdoing with fancy layouts. The color palette was found using color hunt.
Responsiveness is mostly handled with bootstrap, but in some occasions I had to hard code the placements of certain elements using css media queries: 


/* On screens that are 1200px or less, set the background color to red */
@media screen and (max-width: 1200px) {
  body {
    background-color: red;
  }
}

Achieve goal 1: dark mode

It is a good practice to define a color palette using css variables in a separate css file. This variables are then used inside the main css file. The html page loads then the variables and the main css styiling sheet.

To get dark mode woking I created two variables sheets (one for light and one for dark mode). Using javascript it is then possible to define which variables sheet to load in the html page. In javascript a variable defines which theme is currently selected (by default it is light). A switch button is also linked in the javascript. When the user presses the button the theme is changed and it is stored inside the localstorage. In this way the theme will be persistent across all the pages and saved in the browser.
Here is a snippet: 


let theme = localStorage.getItem('theme');
if (theme == null) {
    setTheme('light');
} else
{
    setTheme(theme);
}

// ...

function setTheme(mode) {
    if (mode == 'dark')
    {
        // change stylesheet
        document.getElementById('theme-style').href = '/css/partials/_variables_dark.css';
    } 
    //...
}


Also notice that the images in the main page are not .png or .jpg . Since I knew that I wanted the images to be affected by the theme of the website I converted the .png images into .svg using convertio. Afterwards I used iconmoon to create a font. This font is actually a collection of the images I used in the website and the nice thing is that all css font properties can be used on these images. So changing the color is a matter of using the color property in css.

Achieve goal 2: Dynamic blog posts
I didn't want to add all the information for each blog post manually inside the html. Instead I wanted to have an automatic way of showing the blog posts in the projects section.

This made it also easy to add logic for not showing more than 5 posts in the main page, but showing a show more button which would redirect to a new page with all the projects.

Additionally this approach allows to sort the blog posts in any order using javascript.

Since the scope of this project is not very big I decided to take a simple approach and use a .json file which would contain all the information for every blog post:
BlogPost Id Date Tag Title Description Project Link Image Link
0 10.02.2018 Vision Ray Marcher A fast ray marcher in c /blog/post-ray-march.html /img0.png
1 14.08.2019 Game Mobile Game Building a mobile game from scratch /blog/post-mobile-game.html /img1.png
A javascript file reads all the content using fetch, creates a blogPost object for every entry in the file, saves them into an array and sorts the array from most recent to least recent. Finally based on the filtering option, not more than 5 cards are shown in the projects section.
Achieve goal 3: Filter blog posts
Multiple tag buttons are linked to the script with an event for the click action. This event is handled by filtering the array of blog posts based on the clicked tag and replace the current cards by the filtered content. Filtering becomes as simple as this: 

// blogPosts is the array containing all the blogPost objects
filteredPosts = blogPosts.filter(elem => elem.tag == selectedTag);
/* 
   show is a function that replaces the content of the 
   current cards with 5 elements of the filtered array 
*/
show(filteredPosts);

Finalizing the website

Finally I added some small scripts to have smooth scrolling in the main page when selecting a menu item. This caused some trouble for the active state because I had to activate the buttons not only when clicking but also when scrolling.

I managed to make this work but unfortunately when pressing a menu item there is a small flickering in the active states. Since I don't find this to be a big deal I decided to not spend time to solve it. Funny enough I realised that also some websites using elementor in wordpress have this issue.

This concludes the journey of creating this website in 1 week. I'm happy I decided to work on the website from scratch. It was a very rewarding experience, and I finally can say that I have some idea about web desing.
